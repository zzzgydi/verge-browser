# Verge Browser - 待办事项与技术文档

## 1. 项目愿景回顾

基于 req.md 的核心目标：

1. **自部署 + 简单鉴权**：用户可自部署，配置 admin auth token
2. **管理页面**：Vite + React 实现，admin token 登录后查看所有实例
3. **Agent 友好**：高度封装的鉴权，提供 SDK/CLI，可包装为 skills
4. **人工接管**：VNC 配合 agent，处理扫码登录、账号登录等人工操作
5. **自动化能力**：结合 Playwright 进行自动化操作

## 2. 参考分析: agent-browser

[vercel-labs/agent-browser](https://github.com/vercel-labs/agent-browser) 是值得深入参考的项目，主要差异和借鉴点：

### 2.1 核心差异

| 维度     | agent-browser                | Verge Browser (当前/规划) |
| -------- | ---------------------------- | ------------------------- |
| 运行模式 | 无状态 CLI，每次启动新浏览器 | 持久 Sandbox，可暂停/恢复 |
| 人机协作 | 纯自动化，无 VNC             | VNC 接管为核心特性        |
| 交互方式 | Accessibility Tree + Refs    | GUI 坐标操作              |
| 管理界面 | 无，纯 CLI                   | Web UI 为核心需求         |
| 部署     | 本地 CLI / Docker            | 自托管服务                |

### 2.2 值得借鉴的设计

1. **Semantic Locators** (语义定位)
   - 使用 `role`, `text`, `label` 而非坐标
   - 降低 LLM 幻觉，提高鲁棒性

2. **Snapshot 概念**
   - 返回结构化的页面状态（accessibility tree）
   - 带 `@ref` 标记供后续操作引用

3. **CLI 设计**
   - 命令式交互：`snapshot`, `click @e2`, `find "Submit" click`
   - 直观的 trace/debug 工具

4. **Skills 封装**
   - 将常用操作打包为可复用 skills
   - `.claude-plugin` 直接集成到 Claude

5. **Session 持久化**
   - `state save/load` 保持登录态
   - CDP `connect <port>` 连接现有实例

```python
client = VergeClient(base_url="http://localhost:8000", token="admin-token")

# 创建 sandbox
sandbox = client.create_sandbox(width=1440, height=900)

# 获取 CDP URL 给 Playwright

with sync_playwright() as p:
    browser = p.chromium.connect_over_cdp(sandbox.cdp_url)
    page = browser.new_page()
    page.goto("https://example.com")

# 截图

# GUI 操作
sandbox.click(x=100, y=200)

# VNC 接管

# 清理
sandbox.delete()
```

### 3.4 Phase 4: Playwright 集成示例 (P1)

**目标**: 提供完整的 Playwright + Verge 使用示例

#### 3.4.1 示例代码

```python
# examples/playwright_integration.py
"""
演示如何结合 Verge Browser 和 Playwright 进行自动化操作
包括: 创建 sandbox、连接 CDP、自动化操作、截图、VNC 接管
"""

from playwright.async_api import async_playwright
from verge import VergeClient

async def main():
    client = VergeClient()

    # 1. 创建 sandbox
    sandbox = await client.create_sandbox()
    print(f"Sandbox created: {sandbox.id}")
    print(f"VNC URL: {sandbox.get_vnc_url()}")  # 人工接管入口

    # 2. 连接 Playwright
    async with async_playwright() as p:
        browser = await p.chromium.connect_over_cdp(sandbox.cdp_url)
        context = await browser.new_context()
        page = await context.new_page()

        # 3. 自动化操作
        await page.goto("https://github.com/login")

        # 4. 遇到需要人工干预的环节(如验证码)
        # 打印 VNC URL 给用户
        print("请打开 VNC URL 完成验证码/扫码登录")
        input("完成后按回车继续...")

        # 5. 继续自动化
        await page.goto("https://github.com/settings")

        # 6. 截图验证
        await sandbox.take_screenshot()

    # 7. 清理
    await sandbox.delete()

if __name__ == "__main__":
    import asyncio
    asyncio.run(main())
```

### 4.1 Admin 鉴权设计

```python
# app/auth/admin.py
from fastapi import HTTPException, Header, Depends

class AuthContext:
    subject: str  # 用户标识
    is_admin: bool
    token_type: "jwt" | "admin" | "anonymous"

def get_auth_context(
    authorization: str | None = Header(None),
    x_admin_token: str | None = Header(None),
) -> AuthContext:
    """
    优先级:
    1. Admin Token (如果配置了)
    2. JWT Bearer Token
    3. Anonymous (如果允许)
    """
    ...

def require_admin(auth: AuthContext = Depends(get_auth_context)):
    if not auth.is_admin:
        raise HTTPException(403, "admin required")
    return auth

def require_owner_or_admin(
    sandbox_id: str,
    auth: AuthContext = Depends(get_auth_context),
):
    sandbox = registry.get(sandbox_id)
    if auth.is_admin or sandbox.owner == auth.subject:
        return auth
    raise HTTPException(403, "permission denied")
```

#### 4.4.2 CDP 实现方案

利用 Chrome DevTools Protocol 的 `Accessibility` domain：

```python
# 获取 accessibility tree
await cdp.send("Accessibility.enable")
tree = await cdp.send("Accessibility.getFullAXTree")

# 解析关键字段
# - backendDOMNodeId: 关联 DOM
# - role: 元素角色 (button, link, textbox, etc.)
# - name: 可访问名称
# - value: 当前值
# - bounds: 边界框 (用于计算坐标)
```

#### 4.4.3 Snapshot 数据模型

#### 4.4.4 Semantic Locator 语法

```python
# 基础查找
sandbox.find(role="button", name="Submit")
sandbox.find(text="Sign In")
sandbox.find(label="Email")
sandbox.find(placeholder="Search...")

# 组合条件
sandbox.find(role="button", name=re.compile(r"^Add to"))
sandbox.find(near_text="Total", role="textbox")

# 链式过滤
sandbox.find(role="listitem").find(role="button", name="Delete")
```

#### 4.4.5 与坐标操作的对比

| 场景       | 坐标操作                            | Semantic Locator                               |
| ---------- | ----------------------------------- | ---------------------------------------------- |
| 按钮点击   | `click(x=340, y=210)`               | `find(role="button", name="Submit").click()`   |
| 表单填写   | `click(x=100, y=200); type("text")` | `find(label="Email").fill("user@example.com")` |
| 稳定性     | 分辨率/布局变化会失效               | 基于语义，更鲁棒                               |
| LLM 友好度 | 需要截图 + 视觉推理                 | 直接解析 snapshot                              |
| 速度       | 快（直接执行）                      | 稍慢（需获取 tree）                            |

建议：**混合使用**

- 高频/稳定操作：坐标
- 探索/变化页面：Semantic Locators

| 表单填写   | `click(x=100, y=200); type("text")` | `find(label="Email").fill("user@example.com")` |
| 稳定性     | 分辨率/布局变化会失效               | 基于语义，更鲁棒                               |
| LLM 友好度 | 需要截图 + 视觉推理                 | 直接解析 snapshot                              |
| 速度       | 快（直接执行）                      | 稍慢（需获取 tree）                            |

### 5.3 快速验证点

每个阶段完成后应验证：

| 阶段          | 验证点                                                                                  |
| ------------- | --------------------------------------------------------------------------------------- |
| Admin 鉴权    | 配置 `VERGE_ADMIN_AUTH_TOKEN` 后，只有带正确 token 的请求能查看所有 sandbox             |
| 管理页面      | 能通过 Web UI 完成: 登录 -> 创建 sandbox -> 打开 VNC -> 删除 sandbox                    |
| SDK/CLI       | 能用 SDK 完成: 创建 -> Playwright 连接 -> 截图 -> 删除 的流程                           |
| Accessibility | `verge snapshot` 返回 accessibility tree；`verge find --role button --click` 能正确点击 |
| 示例          | 示例代码可直接运行，无需额外配置                                                        |

## 6. 风险与考虑

1. **安全性**: Admin Token 应强制要求复杂度和 HTTPS
2. **兼容性**: SDK 版本应与 API 版本兼容
3. **部署复杂度**: Web UI 需要构建步骤，考虑提供预构建镜像
4. **维护成本**: SDK/CLI 需要持续维护，建议先实现核心功能
